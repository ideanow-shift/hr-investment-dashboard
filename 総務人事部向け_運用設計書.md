# NOV Talent 総務人事部向け 運用設計書

## 1. システムの目的

NOV Talentは、採用活動を単なる応募管理ではなく「人材投資」として管理するためのダッシュボードです。

総務人事部は、就職フェア、学校接点、学生個別フォロー、採用予算の情報をもとに、以下を判断します。

- 現在の採用進捗は目標に対して順調か
- どのフェアに次年度も投資すべきか
- どの学校との関係強化を優先すべきか
- 採用投資に対してどれだけ成果が出ているか
- 次に取るべき行動は何か

## 2. 利用者と役割

| 役割 | 主な担当 | 操作内容 |
| --- | --- | --- |
| 総務人事部 | 日常運用担当 | ダッシュボードでの学生フォロー更新、面接・フェア・学校状況の確認 |
| 総務人事責任者 | 判断者 | フェア継続判断、学校訪問優先順位、予算配分判断 |
| 経営層 | 閲覧者 | 採用ROI、達成率、重点投資先の確認 |
| システム管理者 | 保守担当 | GAS更新、GitHub Pages更新、Supabase/GAS接続保守、列追加時のメンテナンス |

## 3. 運用で使う画面

| 画面 | 見る目的 | 主な判断 |
| --- | --- | --- |
| サマリー | 採用目標、接触目標、面接目標、予算の進捗確認 | 今月の優先課題 |
| フェア分析 | フェア別ROI、見学率、接触単価の確認 | 次年度参加、見送り、改善対象 |
| 学校分析 | 学校別の接触、見学、面接、内定状況の確認 | 学校訪問の優先順位 |
| 学生フォロー | 個別学生の次アクション確認 | 見学誘導、面接調整、内定後フォロー |
| 面接管理 | 面接実施、診断メモ、最終判断の整理 | 合否判断、再面接判断、総務人事部への共有 |
| データ品質 | 重複、未入力、管理対象外候補の確認 | データ修正、運用ルール見直し |
| 操作履歴 | 誰がいつ何を更新したかの確認 | 保存確認、権限確認、監査 |
| 使い方 | 入力ルールと更新タイミングの確認 | 入力ミス防止 |

## 4. 日常入力の基本方針

現在の日常入力は、原則としてNOV Talentのダッシュボード画面から行います。

学生情報の正本はSupabase側の `talent_students` です。Googleスプレッドシートは、移行期の確認、バックアップ、管理者による一括確認用途として扱います。

フェア実績、学校別分析は、全体傾向を見るための集計情報です。個別学生の状態は必ず学生カードに残します。

### 入力順

1. ダッシュボード右上の `データ更新` を押し、最新状態を確認する
2. `学生フォロー` で学生を追加、または学生カードを更新する
3. 見学、面接、選考結果、次アクションを更新する
4. 必要に応じて `面接管理` に診断メモ、実施者所感、最終判断を残す
5. フェアや学校の傾向は `フェア分析`、`学校分析` で確認する
6. 更新後は `操作履歴` で保存者、保存時刻、結果を確認する

## 5. 更新タイミング

| タイミング | やること | 担当 |
| --- | --- | --- |
| フェア当日 | 接触数、LINE登録数、学生管理の新規行を追加 | 総務人事部 |
| フェア翌営業日 | 見学取得数、次アクション、次アクション日を更新 | 総務人事部 |
| 学校訪問後 | 学校別分析、学生管理の流入元を更新 | 総務人事部 |
| 毎週 | 要フォロー、見学予定、面接予定、内定者を確認 | 総務人事部 |
| 月次 | フェア別投資判断、学校別優先順位、予算消化率を確認 | 総務人事責任者 |
| 四半期 | 次年度のフェア参加候補、学校訪問計画を見直す | 総務人事責任者、経営層 |

## 6. 判断基準

### フェア別投資判断

| 評価 | 目安 | 判断 |
| --- | --- | --- |
| S | 見学率50%以上 | 次年度も優先参加 |
| A | 見学率30%以上 | 継続参加候補 |
| B | 見学率15%以上 | 条件付き継続 |
| C | 見学率1%以上 | 改善して再検討 |
| D | 見学0 | 次回見送り候補 |

### 学校別投資判断

以下の学校を優先候補とします。

- 接触数が多い
- LINE登録数が多い
- 見学数が多い
- 面接数が多い
- 内定数が出ている

特に、見学数と面接数が多い学校は、次回学校訪問や先生との関係強化を優先します。

## 7. 学生フォローの考え方

学生管理では、全員に次アクションを設定します。

| 状態 | 次アクション例 |
| --- | --- |
| LINE登録済、見学未設定 | 見学誘導LINE送信 |
| 見学予定 | 見学前リマインド |
| 見学実施済、面接未設定 | 面接案内 |
| 面接予定 | 面接前確認 |
| 内定 | 内定後フォロー面談 |
| 入社予定 | 入社前フォロー |

次アクション日は、日付が決まっている場合は必ず入力します。未定の場合も、次アクション欄には何をするかを残します。

## 8. スプレッドシートメニュー

Googleスプレッドシート上部の `NOV Talent` メニューは、移行期または管理者作業で使います。日常運用はダッシュボードから行います。

| メニュー | 用途 | 注意 |
| --- | --- | --- |
| 入力規則を再設定 | プルダウン、日付形式、ヘッダー固定を再設定 | データは上書きしない |
| 既存シートを最新化 | 新しい列を追加し、入力規則を再設定 | データは上書きしない |
| 初期サンプルシートを作成 | 初回構築時のサンプル作成 | 既存データを上書きするため通常運用では使わない |

## 9. 注意事項

- 1行目の列名は変更しない
- シート名は変更しない
- 学校名、フェア名、流入元は表記を統一する
- 数字には円記号やカンマを入れない
- 日付はGoogleスプレッドシートの日付形式で入力する
- 同じ学生を複数登録しない。重複が疑われる場合は、削除ではなく `管理対象外` または統合候補として扱う
- 通常運用ではスプレッドシートを直接編集しない
- 学生データを物理削除しない
- 現職者管理、労務、家族、扶養、給与、保険、マイナンバー等のHR秘匿情報はNOV Talent側へ入力しない
- 個人情報を含むため、共有範囲は総務人事部と関係者に限定する

## 10. 引き渡し時に共有するもの

総務人事部へ共有するものは以下です。

- GitHub PagesのダッシュボードURL
- GoogleスプレッドシートURL（移行確認・バックアップ用途）
- この運用設計書
- `スプレッドシート入力ルール.md`（移行期・管理者向け）
- `総務人事部向け_日常運用マニュアル.md`
- `スプレッドシート実データ照合_移行手順.md`
- 「初期サンプルシートを作成」は通常押さない、という注意

## 11. 総務人事部向け説明文

NOV Talentは、採用活動を「人材投資」として見える化するための管理ダッシュボードです。

普段はダッシュボードから学生カードを更新します。保存後、必要に応じてダッシュボード右上の `データ更新` を押すと、最新状況が反映されます。

週次では学生フォロー、月次ではフェア別ROIと学校別投資効果を確認し、次に優先すべきフェア、学校、学生フォローを判断してください。

Googleスプレッドシートは、移行期のデータ照合やバックアップ確認に使います。日常的な学生更新は、原則NOV Talent上で行ってください。
