# リクルート管理から現職者管理への引継ぎ項目

更新日: 2026-07-07

## 目的

リクルート管理システムで内定・入社決定になった学生を、将来の現職者管理へ引き継ぐための項目整理です。

現時点では実装・DB反映は行わず、Core DB / HUB / HR Moduleレビュー前の設計メモとして扱います。

## 基本方針

- 学生の正本は `talent_students.id`
- 社員化後の正本は `employees.id`
- 店舗は `stores.id`
- 部署は `departments.id`
- 役職は `positions.id`
- 職種は `job_types.id`
- 権限は `roles / employee_roles`
- HR秘匿情報はNOV Talent側に保存しない

## 引継ぎタイミング

| タイミング | 内容 | 承認者 |
| --- | --- | --- |
| 内定 | 入社候補者として管理開始 | 総務人事責任者 |
| 承諾 | 入社手続き準備へ移行 | 総務人事部 |
| 入社日確定 | 配属・入社手続きチェック開始 | 総務人事部 |
| 入社完了 | Core DB社員マスタ反映候補 | Core/HUB承認後 |

## リクルート管理から渡す候補項目

| 項目 | 現在の所在 | 将来の扱い | 注意 |
| --- | --- | --- | --- |
| 学生ID | `talent_students.id` | 引継ぎ元IDとして保持 | 社員IDとは別 |
| 氏名 | `talent_students.full_name` | 社員作成時の初期値候補 | 正式表記確認が必要 |
| 学校名 | `talent_students.school_name_snapshot` | 採用経路情報 | 社員マスタ正本にはしない |
| 卒業区分 | `cohort` | 採用年度分類 | 分析用 |
| 性別 | `gender` | 必要な場合のみ参照 | HR秘匿情報扱いに注意 |
| 流入元 | `source_name` | 採用経路 | 分析用 |
| 接触日 | `contact_date` | 採用活動履歴 | 分析用 |
| 選考結果 | `result_status` | 引継ぎ条件 | 内定/承諾相当のみ |
| 希望店舗 | `preferred_store_id`候補 | 配属希望 | stores.id参照 |
| 見学店舗履歴 | `talent_student_store_tours`候補 | 配属検討材料 | stores.id参照 |
| 配属予定店舗 | `expected_store_id`候補 | 入社準備 | stores.id参照 |
| 担当者 | `owner_employee_id` | 引継ぎ担当 | employees.id参照 |
| メモ | `memo` | 必要最小限のみ | 個人情報過多に注意 |

## 現職者管理で新たに扱う候補項目

以下はNOV Talent側で勝手にDB化しません。Core DB / HR Moduleレビュー対象です。

| 項目 | 分類 | 正本候補 |
| --- | --- | --- |
| 入社日 | 労務/社員基本 | Core DB / HR Module |
| 雇用形態 | 社員属性 | employees.employment_type |
| 就労ステータス | 社員属性 | employees.employment_status |
| 部署 | 社員属性 | departments.id |
| 役職 | 社員属性 | positions.id |
| 職種 | 社員属性 | job_types.id |
| 所属店舗 | 社員属性 | stores.id / employee_store_assignments |
| 契約書類 | HR秘匿/書類 | HR Module |
| 給与・保険・税 | HR秘匿 | HR Module |
| 退職手続き | 労務履歴 | HR Module / Core DBレビュー |
| 休職・復職 | 労務履歴 | HR Module / Core DBレビュー |
| 異動・昇格 | 社員履歴 | employee_assignment_histories |

## 画面上の想定

### リクルート管理側

- 学生カードで `内定/承諾/入社日確定` 相当の状態を確認
- `現職者管理へ引継ぎ候補` として表示
- 引継ぎ前に不足項目を確認
- Core DBへ直接反映しない

### 現職者管理側

- 入社手続きチェックリスト
- 配属予定確認
- 必要書類ステータス
- 社員マスタ反映待ちキュー
- 承認ログ

## Coreレビューが必要なもの

- 学生から社員への変換フロー
- `talent_students.id` と `employees.id` の紐付けテーブル
- 入社手続きチェックリストのテーブル化
- 社員マスタ反映待ちキュー
- HR秘匿情報の保存場所
- 承認者、作成者、更新者の `employees.id` 参照

## 今はやらないこと

- `employees` への直接INSERT/UPDATE
- HR秘匿情報のNOV Talent保存
- 現職者管理テーブルのDDL投入
- 入社・退職・休職・異動の本番反映
- Secret/service_role変更
