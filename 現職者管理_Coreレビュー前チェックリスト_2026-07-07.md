# 現職者管理 Coreレビュー前チェックリスト

作成日: 2026-07-07

## 目的

リクルート管理システムで内定・承諾・入社予定になった学生を、将来の現職者管理システムへ安全に引き継ぐための確認リストです。

現時点では社員マスタ更新、DDL投入、Core DB更新は行いません。

## まず画面で確認すること

- 入社手続き候補に対象者が表示される
- 配属希望店舗が入力されている
- 見学店舗履歴が確認できる
- 次アクションと次アクション日が入っている
- 内定後フォローのメモが残っている

## 総務人事部で運用を決めること

- 入社予定日はどのタイミングで確定扱いにするか
- 必要書類チェックをどこまでシステム化するか
- 店舗への共有完了を誰が押すか
- 社員マスタ作成前の承認者を誰にするか
- 入社辞退になった場合の戻し方

## Core DBレビューが必要になるもの

- 入社手続きキューの `talent_` テーブル化
- 学生 `talent_students.id` と社員 `employees.id` の紐付け
- 配属予定店舗 `stores.id` 参照
- 作成者、更新者、承認者の `employees.id` 参照
- 社員マスタ反映待ち、反映済み、差し戻しのステータス設計
- 操作履歴、承認履歴、通知履歴の保存方針

## まだ実行しないこと

- Supabase DDL投入
- RLS / RPC / GRANT変更
- role / employee_roles変更
- public.employees の作成・更新
- stores / departments / positions / job_types の独自マスタ化
- Secret / service_role変更
- 一括UPDATE / DELETE

## 現時点で進めてよいこと

- 画面表示の改善
- 総務人事部へのヒアリング
- 入社手続き候補の不足項目確認
- Claude外部監査packの整備
- Coreレビュー依頼文の準備
