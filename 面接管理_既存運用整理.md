# 面接管理 既存運用整理

作成日: 2026-07-07

## 現在の運用

現在の新卒面接は、以下の流れで実施されています。

1. 面接時に録音
2. 録音内容をNotebookLMへ読み込み
3. NotebookLMに読み込まれている新卒面接マニュアルに沿って診断
4. 診断結果と面接実施者の話し合いで合否判断
5. 総務人事部へ合否を通達
6. NOV Talent上で学生の選考結果を更新

## システム化の基本方針

面接は1名ずつ実施しない場合があるため、学生カードの中だけに閉じ込めず、独立した「面接管理」導線を持つのが望ましいです。

ただし、現時点では面接履歴テーブルの新規DDLは行わず、既存の学生メモ・選考結果・操作履歴を活かしたUI整理に留めます。

## 画面案

```text
リクルート管理システム
├─ 学生フォロー
└─ 面接管理
   ├─ 面接予定
   ├─ 面接実施済み
   ├─ 診断結果入力待ち
   ├─ 合議待ち
   └─ 合否通達済み
```

## 面接管理で見たい項目

- 面接日
- 面接形式
- 対象学生
- 複数学生面接かどうか
- 面接実施者
- NotebookLM診断結果
- 面接実施者コメント
- 総務人事コメント
- 合議結果
- 通達状況
- 次アクション

## 既存データで当面使う項目

| 用途 | 既存項目 |
| --- | --- |
| 学生 | `talent_students.id` |
| 選考結果 | `result_status` |
| 次アクション | `next_action` |
| 次アクション日 | `next_action_date` |
| 担当者 | `owner_employee_id` / 担当者表示 |
| メモ | `memo` |
| 操作履歴 | `talent_operation_logs` |
