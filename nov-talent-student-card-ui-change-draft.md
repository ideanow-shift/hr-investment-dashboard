# NOV Talent 学生カードUI変更ドラフト

作成日: 2026-07-04  
目的: 学生カードの操作項目が多く見える問題に対し、DBや権限を変更せず、画面整理で日常運用を軽くする。

## 変更方針

- すべての項目を一画面に同列表示しない。
- 日常操作は「次に何をするか」を中心にする。
- 管理項目は詳細・管理タブへ移動する。
- 保存ボタンを見失わないようにする。
- 権限がある人だけ管理項目を見やすくする。ただし保存可否の最終判断はGAS/Core DB側で行う。

## タブ / セクション案

### 1. 今日やること

学生カードを開いた時に最初に見せるタブ。

表示項目:
- 現在ステータス
- 次アクション
- 次アクション日
- 直近メモ
- LINE / 見学 / 面接 / 内定の現在位置

操作:
- 「見学予定にする」
- 「面接予定にする」
- 「内定にする」
- 「フォロー完了にする」
- 「次アクションを更新」

### 2. 基本

初回登録・基本確認用。

表示項目:
- 氏名
- 学校
- 卒年 / 学年
- 性別
- 連絡先
- 流入元
- 接触日

### 3. 進捗

採用ファネル上のステータス管理用。

表示項目:
- LINE登録状況
- サロン見学ステータス
- 面接ステータス
- 選考結果
- 内定・入社ステータス

補足:
- ステータス矛盾がある場合だけ赤い警告を表示する。
- 通常時は説明文を増やしすぎない。

### 4. 見学・店舗

店舗希望と見学履歴用。

表示項目:
- 配属希望店舗 第1希望
- 配属希望店舗 第2希望
- 配属希望店舗 第3希望
- 見学店舗履歴

操作:
- 希望店舗を保存
- 見学履歴を追加

### 5. メモ・履歴

人事メモと履歴確認用。

表示項目:
- メモ
- フォロー履歴
- 操作履歴の要約

### 6. 管理

管理者向け。初期表示では目立たせない。

表示項目:
- 管理状態
- 学生コード
- 作成者
- 更新者
- Core / Supabase連携ID
- LSTEP連携情報
- データ品質警告

## 固定保存ボタン案

### PC

- 学生カード下部に常時表示。
- タブを移動しても「保存」「閉じる」が見える状態にする。
- 保存成功後は「保存しました。データ更新済みです」と短く表示。

### スマホ

- 画面下部に固定。
- ボタンは2つまで。
  - 保存
  - 閉じる
- エラーはボタン直上に赤文字で表示。

## 次アクションを目立たせる案

学生カード上部に小さなアクション帯を置く。

例:

```text
次アクション: 見学リマインド
期限: 2026/07/05
[完了にする] [日程変更] [メモ追加]
```

未設定の場合:

```text
次アクションが未設定です
[見学誘導] [面接調整] [内定後フォロー]
```

## 未入力・不整合の見せ方

- 必須不足は赤ではなく黄で注意表示。
- 保存できない致命的エラーだけ赤。
- ステータス矛盾は「修正候補」ボタンを出す。

例:

```text
注意: 面接ステータスが未設定ですが、選考結果が合格になっています。
[面接 実施済みにする]
```

## 青柳さん向けテスト観点

1. HUBからNOV Talentを開く。
2. 学生カードを開く。
3. 次アクションを変更して保存できるか。
4. 配属希望店舗を変更して保存できるか。
5. 見学履歴を追加して保存できるか。
6. 保存後にカードが固まらないか。
7. どの項目を触ればよいか迷わないか。
8. 管理情報が日常操作の邪魔になっていないか。
