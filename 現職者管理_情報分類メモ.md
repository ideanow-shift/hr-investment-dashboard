# 現職者管理 情報分類メモ

作成日: 2026-07-07

## 目的

リクルート管理システムから、将来的に現職者管理システムへ拡張する前に、扱う情報の境界を整理する。

現職者管理は、入社手続き、異動、休職、復職、退職などを扱う可能性がある。
ただし、HR秘匿情報やCore DBの社員正本更新に近いため、NOV Talent側で安易に独自実装しない。

## 基本方針

- 社員正本は `public.employees.id`
- 店舗正本は `public.stores.id`
- 部署正本は `public.departments.id`
- 役職正本は `public.positions.id`
- 職種正本は `public.job_types.id`
- 権限正本は `public.roles` / `public.employee_roles`
- HR秘匿情報はNOV Talent側へ持ち込まない
- Core DB反映は承認制にする
- 最初は「反映待ち」「確認待ち」「承認待ち」のキューとして設計する

## NOV Talent / リクルート管理側で持ってよい候補

| 情報 | 扱い | 理由 |
| --- | --- | --- |
| 学生ID | `talent_students.id` | 採用候補者の正本 |
| 学生氏名 | 表示用 | 採用候補者管理に必要 |
| 学校 | talent側学校IDまたはsnapshot | 採用経路分析に必要 |
| 選考結果 | talent側 | 採用判断に必要 |
| 面接メモ | talent側 | 採用判断・申し送りに必要 |
| 希望店舗 | stores.id参照 | 配属相談の材料 |
| 見学店舗履歴 | stores.id参照 | 配属相談の材料 |
| 採用担当者 | employees.id参照 | 監査・引き継ぎに必要 |
| 更新者 | employees.id参照 | 操作履歴に必要 |

## 現職者管理で扱うがレビュー必須の候補

| 情報 | レビュー先 | 注意 |
| --- | --- | --- |
| 入社手続きステータス | Core DB番人 / HUB HR | 社員化前の中間状態 |
| 配属予定店舗 | Core DB番人 / HUB Core | stores.id参照 |
| 雇用形態 | Core DB番人 / HUB HR | employees.employment_typeとの境界確認 |
| 職種 | Core DB番人 / HUB Core | job_types.id参照 |
| 部署 | Core DB番人 / HUB Core | departments.id参照 |
| 役職 | Core DB番人 / HUB Core | positions.id参照 |
| 異動手続き | Core DB番人 / HUB HR | assignment historiesとの境界 |
| 休職・復職手続き | Core DB番人 / HUB HR | HR秘匿情報に接近 |
| 退職手続き | Core DB番人 / HUB HR | employment_status更新に接近 |
| 承認者 | Core DB番人 / HUB Core | employees.id参照 |
| 承認ログ | Core DB番人 | 監査ログ方針が必要 |

## NOV Talent側に持ち込まない情報

以下はHR秘匿情報または高リスク情報のため、NOV Talent側の通常DB・フロント・CSVへ持ち込まない。

- マイナンバー
- 給与
- 銀行口座
- 扶養家族
- 住所詳細
- 通勤経路詳細
- 保険情報
- 税情報
- 健康情報
- 休職理由の詳細
- 診断書
- 契約書ファイル本体
- LINE WORKS Secret
- Supabase service_role key
- LSTEP APIキー

## 入社手続きで必要になりそうなチェック項目

まずはDB化せず、チェックリスト案として扱う。

- 入社日確認
- 配属予定店舗確認
- 雇用形態確認
- 職種確認
- 役職確認
- 必要書類案内
- 契約書作成依頼
- 社員番号発番待ち
- 社員マスタ反映待ち
- HUBログイン準備
- LINE WORKS準備
- 初日オリエンテーション案内

## 退職手続きで必要になりそうなチェック項目

まずはDB化せず、チェックリスト案として扱う。

- 退職申出日
- 最終出勤日
- 退職予定日
- 有給消化確認
- 貸与物返却確認
- アカウント停止依頼
- 社保・労務確認
- 退職書類案内
- 社員マスタ反映待ち

## 異動手続きで必要になりそうなチェック項目

まずはDB化せず、チェックリスト案として扱う。

- 異動予定日
- 異動元店舗
- 異動先店舗
- 役職変更有無
- 職種変更有無
- 給与・契約影響有無
- 本人連絡
- 店長連絡
- 社員マスタ反映待ち

## 休職・復職手続きで必要になりそうなチェック項目

まずはDB化せず、チェックリスト案として扱う。

- 休職開始予定日
- 復職予定日
- 休職種別
- 本人連絡
- 店舗連絡
- 社保・労務確認
- 復職面談予定
- 社員マスタ反映待ち
