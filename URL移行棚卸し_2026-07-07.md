## Migration Risks

### 1. GitHub Pages URL

`ideanow-shift.github.io` から `idea-nov` へnamespace変更する場合、HUBカード、社内共有リンク、ブラウザブックマークが影響を受ける。

対応方針:

- 変更前にHUB側 `portal_apps.url` の棚卸しが必要。
- 旧URLから新URLへの案内期間を設ける。
- GitHub Pages反映確認後にHUBカードを切り替える。

### 2. GAS API URL

GAS WebアプリURLはGitHub Pagesのnamespace変更とは別物。
repo renameやPages URL変更だけでは、GAS URLは変わらない。

対応方針:

- `app.js` の `GAS_API_URL` は変更しない。
- GASを新デプロイした場合のみURL確認が必要。
- 新旧デプロイが混在しないよう、デプロイ日時とURLを作業ログに残す。

### 3. HUB context

HUBからの遷移URLが変わる場合、`hub_context` の受け取り処理が動くか確認が必要。

対応方針:

- URL変更前に、HUB経由で社員名、社員番号、roleKeysが表示されるか確認する。
- 保存系はフロントだけで権限判定せず、GAS / Core DB側の再判定を維持する。

### 4. Supabase / Core DB

Supabase project URLやservice_role keyは変更しない。
URL移行に伴ってSecretを触る必要はない。

対応方針:

- Script Propertiesは変更しない。
- Core DBのDDL、RLS、GRANT、role変更はしない。
