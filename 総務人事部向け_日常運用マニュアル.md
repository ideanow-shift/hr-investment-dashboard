# NOV Talent 総務人事部向け 日常運用マニュアル

この資料は、総務人事部が NOV Talent を日常業務で使うための運用ルールです。

NOV Talent は、採用活動を単なる応募管理ではなく「人材投資」として判断するためのダッシュボードです。学生接点、フェア費用、学校別成果、個別フォロー状況を更新し、次に取るべき行動を判断します。

## 1. 基本方針

- 学生情報は、原則としてダッシュボードの「学生フォロー」画面から追加・更新します。
- スプレッドシートを直接編集するのは、移行作業、集計値の一括修正、管理者対応時に限定します。
- 学生データは削除しません。対象外にする場合は `管理状態` を `管理対象外` にします。
- 更新内容は `操作履歴` シートに自動記録されます。
- 入力に迷った場合は、ダッシュボード内の「使い方」タブを確認します。

## 2. 毎日やること

### 学生フォロー確認

1. ダッシュボードを開きます。
2. 右上の `データ更新` を押します。
3. `学生フォロー` タブを開きます。
4. `要フォロー`、`見学予定`、`面接予定` を確認します。
5. 対応した学生のステータス、次アクション、次アクション日、メモを更新します。

### 新規学生追加

1. `学生フォロー` タブを開きます。
2. 対象の区分を選びます。
   - `27卒`
   - `28卒`
   - `サロン実習`
3. `学生追加` を押します。
4. 氏名、学校名、性別、学年、流入元、接触日、LINE登録状況を入力します。
5. 保存後、一覧とスプレッドシートに反映されたことを確認します。

## 3. 毎週やること

### フェア・学校の見直し

1. `フェア分析` タブで、見学率と評価を確認します。
2. D評価、見学取得0、費用が高く成果が低いフェアを確認します。
3. `学校分析` タブで、有望度が高い学校を確認します。
4. 次回訪問・フェア参加・LINE誘導改善の候補を整理します。

### 次アクション整理

1. `学生フォロー` タブの次アクション一覧を確認します。
2. 日付が空欄の次アクションを優先して補完します。
3. 見学予定、面接予定、内定後フォローの抜け漏れを確認します。

## 4. 月次でやること

### 人材投資状況の確認

1. `全体サマリー` タブを開きます。
2. 採用目標人数、現在の内定数、達成率、接触人数目標、面接成約目標を確認します。
3. 使用済み予算、1人あたり採用投資額を確認します。
4. フェア別・学校別の投資判断を次月の行動計画に反映します。

### データ品質確認

1. `操作履歴` シートで直近の更新を確認します。
2. `管理対象外` フィルターで対象外学生を確認します。
3. 重複登録やステータス矛盾がないか確認します。
4. 学校名・フェア名・流入元の表記ゆれを確認します。

## 5. 入力時の注意

### 必須項目

- 氏名
- 学校名
- 流入元
- 接触日
- LINE登録状況

### ステータスの考え方

- 見学予定・面接予定がある場合は、次アクション日を入れます。
- 内定・承諾にする場合は、面接ステータスを `実施済` にします。
- 入社予定・入社済にする場合は、内定ステータスを `内定` または `承諾` にします。
- 不合格の場合、内定ステータスは `未定` または `辞退` にします。

### 管理対象外の使い方

次のような場合は、削除せず `管理状態` を `管理対象外` にします。

- 誤登録した学生
- 重複登録の片方
- 今期の管理対象から外す学生
- 分析には残したいが、通常フォロー一覧から外したい学生

## 6. スプレッドシートを直接触る場合

通常はダッシュボードから入力します。ただし、以下の場合はスプレッドシートを直接編集しても構いません。

- 既存データの一括移行
- フェア実績の一括入力
- 学校別分析の集計値更新
- 管理者による列・入力規則のメンテナンス

直接編集した後は、ダッシュボード右上の `データ更新` を押してください。

## 7. トラブル時

### 保存できない

- 赤文字のエラーメッセージを確認します。
- 氏名・学校名の重複がないか確認します。
- ステータス矛盾がないか確認します。
- 見学予定・面接予定なのに次アクション日が空欄でないか確認します。

### ダッシュボードに反映されない

1. 右上の `データ更新` を押します。
2. ブラウザを `Ctrl + F5` で再読み込みします。
3. スプレッドシート側で対象行が更新されているか確認します。
4. Apps Script が最新版でデプロイされているか確認します。

### 操作履歴が残らない

- Apps Script の `Code.gs` が最新版か確認します。
- Webアプリが新しいバージョンでデプロイされているか確認します。
- `操作履歴` シートが存在するか確認します。
