# リクルート管理システム 総務人事部向け 日常運用マニュアル

この資料は、総務人事部が `リクルート管理システム` を日常業務で使うための運用ルールです。

リクルート管理システムは、採用活動を単なる応募管理ではなく「人材投資」として判断するためのダッシュボードです。学生接点、フェア費用、学校別成果、個別フォロー状況を更新し、次に取るべき行動を判断します。

将来的には、総務人事部が扱う上位の `総務人事管理システム` として、以下の2領域に分けて運用します。

```text
総務人事管理システム
├─ リクルート管理システム
└─ 現職者管理システム
```

現時点で日常運用するのは `リクルート管理システム` です。`現職者管理システム` は、入社手続き、異動、休職、復職、退職などを扱う将来領域として設計中です。

## 1. 基本方針

- 学生情報は、原則としてダッシュボードの「学生フォロー」画面から追加・更新します。
- スプレッドシートを直接編集するのは、移行確認、バックアップ確認、管理者対応時に限定します。
- 学生データは削除しません。対象外にする場合は `管理状態` を `管理対象外` にします。
- 更新内容はダッシュボードの `操作履歴` に自動記録されます。
- 入力に迷った場合は、ダッシュボード内の「使い方」タブを確認します。

## 2. 毎日やること

### 学生フォロー確認

1. ダッシュボードを開きます。
2. 右上の `データ更新` を押します。
3. `学生フォロー` タブを開きます。
4. `要フォロー`、`見学予定`、`面接予定` を確認します。
5. 対応した学生のステータス、次アクション、次アクション日、メモを更新します。

### 新規学生追加

1. `学生フォロー` タブを開きます。
2. 対象の区分を選びます。
   - `27卒`
   - `28卒`
   - `サロン実習`
3. `学生追加` を押します。
4. 氏名、学校名、性別、学年、流入元、接触日、LINE登録状況を入力します。
5. 保存後、一覧とSupabase上のデータに反映されたことを確認します。

## 3. 毎週やること

### フェア・学校の見直し

1. `フェア分析` タブで、見学率と評価を確認します。
2. D評価、見学取得0、費用が高く成果が低いフェアを確認します。
3. `学校分析` タブで、有望度が高い学校を確認します。
4. 次回訪問・フェア参加・LINE誘導改善の候補を整理します。

### 次アクション整理

1. `学生フォロー` タブの次アクション一覧を確認します。
2. 日付が空欄の次アクションを優先して補完します。
3. 見学予定、面接予定、内定後フォローの抜け漏れを確認します。

## 4. 月次でやること

### 人材投資状況の確認

1. `全体サマリー` タブを開きます。
2. 採用目標人数、現在の内定数、達成率、接触人数目標、面接成約目標を確認します。
3. 使用済み予算、1人あたり採用投資額を確認します。
4. フェア別・学校別の投資判断を次月の行動計画に反映します。

### データ品質確認

1. ダッシュボードの `操作履歴` タブで直近の更新を確認します。
2. `管理対象外` フィルターで対象外学生を確認します。
3. 重複登録、管理対象外にすべき学生、次アクション日の抜けがないか確認します。
4. 学校名・フェア名・流入元の表記ゆれを確認します。

## 5. 入力時の注意

### 必須項目

- 氏名
- 学校名
- 流入元
- 接触日
- LINE登録状況

### ステータスの考え方

- 見学予定・面接予定がある場合は、次アクション日を入れます。
- 選考結果は `未定`、`合格`、`条件付き合格`、`再面接`、`不合格`、`辞退` から選びます。
- 合否判断は、面接管理のNotebookLM診断要約、面接実施者の所感、話し合いメモを確認して更新します。
- 内定後の入社手続き、退職、休職、異動などの労務手続きは、将来の現職者管理側で扱う想定です。

### 管理対象外の使い方

次のような場合は、削除せず `管理状態` を `管理対象外` にします。

- 誤登録した学生
- 重複登録の片方
- 今期の管理対象から外す学生
- 分析には残したいが、通常フォロー一覧から外したい学生

## 6. スプレッドシートを直接触る場合

通常運用では、原則としてダッシュボードから入力します。スプレッドシートは移行期の確認・バックアップ用途として扱います。

以下の場合のみ、管理者判断で直接編集します。

- 既存データの一括移行
- フェア実績の一括確認
- 学校別分析の集計値確認
- 管理者による列・入力規則のメンテナンス

直接編集した後は、ダッシュボード右上の `データ更新` を押してください。

## 7. トラブル時

### 保存できない

- 赤文字のエラーメッセージを確認します。
- 氏名・学校名の重複がないか確認します。
- 見学予定・面接予定なのに次アクション日が空欄でないか確認します。
- HUBから開いているか、操作権限がある社員でログインしているか確認します。

### ダッシュボードに反映されない

1. 右上の `データ更新` を押します。
2. ブラウザを `Ctrl + F5` で再読み込みします。
3. 操作履歴に対象操作が残っているか確認します。
4. Apps Script が最新版でデプロイされているか確認します。

### 操作履歴が残らない

- Apps Script の `Code.gs` が最新版か確認します。
- Webアプリが新しいバージョンでデプロイされているか確認します。
- HUBから開き、社員ID付きで保存しているか確認します。
