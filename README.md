# リクルート管理システム / NOV Talent

IDEA NOVグループ向けの「人材投資管理システム」から発展した、総務人事部向けのリクルート管理システムです。

このアプリは、単なる求人管理ではなく、採用活動を「人材投資」として可視化するためのダッシュボードです。総務人事部が、求人費・就職フェア・学校接点・学生接触データをもとに、採用ROI、学校別投資効果、フェア別投資判断、次に取るべき行動を確認できることを目的としています。

2026-07-07時点では、実装・運用の中心は `リクルート管理システム` です。
将来的には、総務人事部が扱う上位システムとして以下の2領域に整理する構想です。

```text
総務人事管理システム
├─ ① リクルート管理システム
└─ ② 現職者管理システム
```

現職者管理システムは、入社手続き、異動、休職、復職、退職などを扱う構想です。
ただし、HR秘匿情報やCore DB更新に関わるため、現時点では設計・レビュー対象であり、本番実装やDB変更は行いません。

## 起動方法

1. `hr-investment-dashboard-gas-retry` フォルダを開きます。
2. `index.html` をブラウザで開きます。

ローカルサーバーを使う場合は、以下のように起動できます。

```bash
python -m http.server 8000
```

その後、ブラウザで `http://localhost:8000/` を開きます。

## 現在の状態

現在はHTML / CSS / JavaScriptで動作するGitHub Pagesアプリです。
データ取得・保存はGAS backendを経由し、Supabaseを正本として扱う段階へ移行しています。
Googleスプレッドシートは移行確認・バックアップ用途として残しています。

- Firebase接続なし
- GAS URL未設定時はサンプルデータで表示
- GAS URL設定時はGAS backendからJSONPでデータ取得
- 画面右上の `データ更新` ボタンでGAS / Supabaseから再取得
- ページを開いている間は24時間ごとに自動再取得
- GitHub Pagesで公開可能
- PC / スマホ対応
- NOV HUBに組み込める前提のカード型ダッシュボードUI
- HUB contextを受け取り、操作履歴に社員IDを残す方針
- service_role keyやSecretはフロントに出さない

## Googleスプレッドシート + GAS 接続手順

混線防止のため、今回は `clasp` を使いません。必ず1つのGoogleスプレッドシートに紐づいたApps Scriptだけを使います。

### 1. スプレッドシートを作成

Googleドライブで新しいスプレッドシートを作成します。

推奨名:

```text
人材投資管理システム データ
```

### 2. Apps Scriptを開く

スプレッドシート上部メニューから以下を開きます。

```text
拡張機能 → Apps Script
```

この方法で開いたApps Scriptだけを使ってください。

### 3. Code.gsを貼り付け

このフォルダ内の以下を開き、内容をApps Scriptの `Code.gs` に貼り付けます。

```text
gas/Code.gs
```

### 4. サンプルシートを作成

Apps Script画面上部の関数選択で以下を選び、実行します。

```text
setupSampleSheets
```

初回は権限承認が必要です。実行後、スプレッドシートに以下4シートが作成されます。

- 年度設定
- フェア実績
- 学校別分析
- 学生管理

`学生管理` シートは、学生個別のフォロー状況を管理するための入力シートです。

`setupSampleSheets` を実行すると、1行目固定、表示形式、学生管理シートのステータス列プルダウンも自動設定されます。

注意: 既存データが入ったシートで `setupSampleSheets` を再実行すると、サンプルデータで上書きされます。本番運用開始後は、必要な場合だけ実行してください。

既存シートに新しい列だけ追加したい場合は、`setupSampleSheets` ではなく以下を実行してください。

```text
upgradeExistingSheetsToLatestSchema
```

この関数は、既存データを保持したまま `年度設定` に `接触人数目標` と `面接成約目標`、`学生管理` に `性別` を追加し、入力規則と表示形式を再設定します。

Apps Scriptを最新版にした後、スプレッドシートを再読み込みすると上部に `NOV Talent` メニューが表示されます。

- `入力規則を再設定`: プルダウン、日付形式、ヘッダー固定を再設定します。
- `既存シートを最新化`: 既存データを保持したまま最新列を追加します。
- `初期サンプルシートを作成`: 初回作成用です。既存データはサンプルで上書きされます。

移行期に残っている主な列:

```text
学生ID / 氏名 / 性別 / 学校名 / 学年 / 流入元 / 接触日 / LINE登録 /
見学ステータス / 面接ステータス / 選考結果 / 内定ステータス /
入社予定 / 担当者 / 次アクション / 次アクション日 / メモ
```

`内定ステータス` と `入社予定` は、既存データ互換・移行確認用の列です。日常運用では、NOV Talentの学生カードから `LINE登録`、`サロン見学`、`応募`、`面接`、`選考結果`、`次アクション` を更新します。

最新版では、以下の列も追加されています。

```text
年度設定: 接触人数目標
年度設定: 面接成約目標
学生管理: 性別
```

日常運用時の入力ルール、推奨ステータス、更新タイミングは以下を参照してください。

```text
アプリ内の「使い方」タブ
```

補助資料として、移行期・管理者向けのスプレッドシートルールを `スプレッドシート入力ルール.md` にも保管しています。

移行データを入れた場合、以下のシートが存在するとダッシュボードの学生フォロー画面で切り替えて確認できます。

```text
学生管理_27卒
学生管理_28卒
学生管理_サロン実習
学生管理_全件参考
```

総務人事部へ引き渡す際は、以下も共有してください。

```text
総務人事部向け_運用設計書.md
総務人事部向け_日常運用マニュアル.md
スプレッドシート入力ルール.md
```

### 5. Webアプリとしてデプロイ

Apps Script右上の `デプロイ` から新しいデプロイを作成します。

設定:

```text
種類: ウェブアプリ
実行するユーザー: 自分
アクセスできるユーザー: 全員
```

デプロイ後に表示されるWebアプリURLをコピーします。

### 6. JSONP接続確認

コピーしたURLの末尾に以下を付けてブラウザで開きます。

```text
?callback=testCallback
```

成功すると以下のように表示されます。

```javascript
testCallback({"config": ...});
```

通常のJSONだけが表示される場合は、Apps Scriptの `Code.gs` がJSONP対応版に更新されていません。

### 7. app.jsへURL設定

`app.js` の先頭にある以下へ、WebアプリURLを貼り付けます。

```javascript
const GAS_API_URL = "https://script.google.com/macros/s/XXXXXXXX/exec";
```

接続成功時は、画面右上のバッジが以下になります。

```text
● GAS Connected
v0.2
```

失敗時は以下のまま、サンプルデータで表示されます。

```text
Sample Data
v0.1
```

LSTEP本接続を再開する段階では、`supabase/talent_lstep_integration_verify.sql` をSQL Editorで実行し、テーブル、RLS、service_role権限、制約、現在件数を再確認してください。制作会社への確認用には、画面の「LSTEP連携状況」から「連携テンプレート」CSVを出力できます。確認依頼文は `LSTEP制作会社_確認依頼.md` にあります。
