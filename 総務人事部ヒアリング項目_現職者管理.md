# 総務人事部ヒアリング項目: 現職者管理

作成日: 2026-07-07

## 目的

リクルート管理システムの次工程として、現職者管理システムを検討するためのヒアリング項目です。

現職者管理は、入社手続き、異動、休職、復職、退職などを扱う可能性があります。
ただしHR秘匿情報や社員マスタ更新に関わるため、まずは業務の流れと確認項目を整理します。

## 1. 入社手続き

確認したいこと:

- 内定から入社手続き開始に進む判断基準
- 入社手続きで現在使っているチェックリスト
- 入社日確定までの流れ
- 配属店舗の決め方
- 雇用形態、職種、役職の決め方
- 社員番号の発番タイミング
- HUBログイン準備のタイミング
- LINE WORKSなどアカウント準備のタイミング
- 店舗への連絡タイミング
- 入社前に本人へ送る案内

システム化候補:

- 入社手続き開始
- 必要書類確認
- 配属予定確認
- アカウント準備確認
- 社員マスタ反映待ち
- 完了

## 2. 異動手続き

確認したいこと:

- 異動申請は誰が起点になるか
- 異動元、異動先の承認者
- 異動日確定までの流れ
- 異動に伴う役職変更の有無
- 異動に伴う雇用条件変更の有無
- 本人への通知方法
- 店舗への通知方法
- 社員マスタ更新のタイミング

システム化候補:

- 異動予定登録
- 異動元確認
- 異動先確認
- 本人通知
- 社員マスタ反映待ち
- 完了

## 3. 休職・産休・育休・復職

確認したいこと:

- 休職種別の分類
- 産休・育休の開始日、終了予定日
- 復職予定日の確認方法
- 店舗への連絡方法
- 社保・労務確認が必要なタイミング
- 復職面談の有無
- 社員マスタの就労ステータス更新タイミング

注意:

- 休職理由の詳細、診断書、健康情報はHR秘匿領域です。
- NOV Talent側に詳細本文やファイルを保存しない方針で確認します。

システム化候補:

- 休職予定登録
- 休職開始確認
- 復職予定確認
- 復職面談予定
- 社員マスタ反映待ち
- 完了

## 4. 退職手続き

確認したいこと:

- 退職申出の受付方法
- 最終出勤日、退職日の確定方法
- 有給消化確認
- 貸与物返却確認
- アカウント停止依頼
- 店舗への連絡
- 社保・労務確認
- 退職書類案内
- 社員マスタ反映タイミング

システム化候補:

- 退職予定登録
- 最終出勤日確認
- 貸与物返却確認
- アカウント停止依頼
- 社員マスタ反映待ち
- 完了

## 5. 承認フロー

確認したいこと:

- どの手続きに承認が必要か
- 承認者は総務人事部長か、経営層か、店舗責任者か
- 差戻しが必要か
- 承認後に社員マスタへ反映するか
- 承認ログをどこまで残すか

候補:

- 下書き
- 確認待ち
- 差戻し
- 承認待ち
- 承認済
- 社員マスタ反映待ち
- 完了

## 6. 画面導線

確認したいこと:

- リクルート管理と同じ画面に置くべきか
- HUBの別カードにするべきか
- 総務人事部だけ見える画面にするべきか
- 店長が見える範囲はあるか
- 本人が確認する導線は必要か

## 7. まず作るなら

最初に作る候補:

1. 入社手続きチェックリスト
2. 社員マスタ反映待ち一覧
3. 退職手続きチェックリスト
4. 異動手続きチェックリスト

ただし、DB実装はCore DB / HUB HRレビュー後に行います。
