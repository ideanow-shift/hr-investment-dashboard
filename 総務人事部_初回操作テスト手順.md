# 総務人事部 初回操作テスト手順

## 目的

総務人事部メンバーが、リクルート管理システムを日常業務で使えるか確認する。
初回テストでは、機能追加よりも「迷わず操作できるか」「保存できるか」「数値が合っているか」を優先する。

## テスト前提

- NOV HUBからリクルート管理システムを開く。
- 右上にHUB社員名が表示されていることを確認する。
- 右上が `GAS Connected v0.2` になっていることを確認する。
- 直開きでは保存できないため、保存テストは必ずHUB経由で行う。

## 1. ログイン・表示確認

確認すること:

- HUBカード表記とシステム内タイトルが `リクルート管理システム` で揃っている。
- スマホでも開ける。
- 読み込み中の数値が実データのように見えない。
- `データ更新` を押してエラーが出ない。

記録すること:

- 開くまでに時間がかかったか。
- スマホで表示が崩れた箇所があるか。
- 右上のHUB社員名が正しいか。

## 2. 学生フォロー確認

確認すること:

- 学生フォロータブを開ける。
- 27卒 / 28卒 / サロン実習を切り替えられる。
- 氏名・学校名で検索できる。
- 要フォロー、未完了フォロー、内定者の数字が違和感ない。

テスト操作:

1. 任意の学生カードを開く。
2. 次アクションを入力する。
3. 次アクション日を入力する。
4. 保存する。
5. データ更新後も内容が残っているか確認する。

記録すること:

- 保存できたか。
- 保存後に赤いエラーが出たか。
- 項目が多くて迷った箇所があるか。

## 3. 選考ステータス確認

確認すること:

- ライン登録、サロン見学、応募、選考結果を更新できる。
- 選考結果の候補が運用に合っている。
- 内定者数、合格数が実データと合っている。

テスト操作:

1. テスト対象の学生を1名決める。
2. 選考結果を変更する。
3. 保存する。
4. データ更新する。
5. サマリー数値が想定どおり変わるか確認する。

注意:

- 実データを壊さないため、テスト対象は事前に決める。
- 誤って変更した場合は、すぐ元に戻して保存する。

## 4. 面接管理確認

確認すること:

- 面接管理タブを開ける。
- 面接予定者、再面接、通達待ちを確認できる。
- NotebookLM診断結果や面接者所感を残す運用に合いそうか確認する。

記録すること:

- 面接が1名ずつではない場合に見づらい箇所があるか。
- 複数名面接の記録方法として不足する項目があるか。
- 診断結果、最終判断、総務人事への通達状態をどこに残すとよいか。

## 5. フェア・学校分析確認

確認すること:

- フェア分析でROIランキングが見られる。
- フェア詳細が開ける。
- 学校分析で詳細ボタンが反応する。
- CSV出力できる。

記録すること:

- 総務人事部が実際に見たい列が不足しているか。
- フェア追加に必要な項目が不足しているか。
- 学校訪問管理として別ページ化したい項目があるか。

## 6. 現職者管理プレビュー確認

確認すること:

- 現職者管理タブが表示される。
- 入社手続き候補が表示される。
- 社員マスタを更新しない設計プレビューであることが分かる。

記録すること:

- 入社手続き候補に表示したい追加情報があるか。
- 入社手続き、異動、休職・復職、退職で最初に管理したい項目は何か。
- この画面から社員マスタへ反映する前に、誰の承認が必要か。

## 7. フィードバック分類

フィードバックは以下のどれかに分類する。

- 権限 / ログイン
- 保存エラー
- データ不整合
- 学生カード編集
- 面接管理
- フェア / 学校分析
- 現職者管理
- スマホ操作
- 表記 / 文言

## 完了条件

- 総務人事部の主要メンバーがHUBから開ける。
- 学生カードを保存できる。
- 実データの主要数値が一致している。
- 操作しづらい箇所をフィードバック回収表へ記録できている。

## まだ行わないこと

- Supabaseの直接編集
- スプレッドシートの手作業大量修正
- 物理削除
- 社員マスタへの直接反映
- LSTEP本接続
- Secret / service_role変更
