# LSTEP保留中の求人側優先タスク

作成日: 2026-07-07

## 前提

求人領域のLSTEP本接続は、LINE公式アカウント側の現行運用との兼ね合いにより約2か月後まで保留する。

この期間は、LSTEP API/Webhook/自動同期の実装ではなく、リクルート管理システムとして日常運用を安定させる作業を優先する。

## 今進めること

### 1. 学生カードの操作性改善

- 毎日触る項目を上部へ寄せる
- 初回入力項目、管理項目、表示専用項目を分離する
- 次アクションと保存ボタンを見つけやすくする
- スマホでも入力しやすい余白とボタン配置にする

### 2. 総務人事部フィードバック対応

- 保存エラーの再発確認
- 学校詳細ボタンなど、クリックしても動かない箇所の確認
- ステータス項目の文言整理
- 重複学生の安全な非表示/統合候補運用の整理

### 3. 面接実施フローの整理

- 面接実施は学生個別カードだけに閉じず、面接タブとして扱う
- 複数名面接にも対応できる運用メモを作る
- NotebookLM診断結果は、当面はメモ/履歴として保存する
- 構造化DB化する場合は、Core DB番人レビューへ回す

### 4. フェア・学校運用の整理

- フェア追加項目の設計案を固める
- 学校訪問管理ページの項目案を固める
- ただし新規テーブルやDDLは実行しない

### 5. LSTEP再開準備

- 未紐付け学生のCSV出力
- 制作会社へ確認する項目の維持
- 連携テンプレートの確認
- 本接続再開時のCoreレビュー対象を明確にする

## 今は進めないこと

- LSTEP API/Webhook/自動同期の本接続
- Secret/APIキーの追加
- Edge Function deploy
- DDL / RLS / GRANT / RPC
- Core DB正本変更
- role / employee_roles変更
- 実データ一括更新

## 判断が必要になったら戻すもの

- 面接履歴の構造化テーブル
- 学校訪問管理をDB正本化する場合
- フェア項目をSupabaseへ追加する場合
- LSTEP接続を再開する場合
- HUB通知やLINE WORKS通知を追加する場合
